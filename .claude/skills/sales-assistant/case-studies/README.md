# Case Studies — реальные продуктовые базы

Эта папка содержит **готовые продуктовые базы** для разных проектов. Каждая подпапка — отдельный продукт со своим оффером, ценами, сегментами ЦА, кейсами, скриптами оплаты.

## Зачем это
- **Быстрый старт** под конкретный продукт без сборки с нуля
- **Эталон структуры** — посмотри как организован kovcheg/ и сделай аналогично для своего
- **База примеров** — реальные цитаты из закрытых сделок, не выдуманные

## Текущие продукты

### 📚 [kovcheg/](./kovcheg/) — AI-Ковчег
8-недельная программа AI-автоматизации. Тарифы $450 / $1000 / $3500. Методология КВП + 2 голоса (Анастасия эмпатичная / Владислав деловой). Внутри:
- `PRODUCT_OVERVIEW.md` — пакеты, цены, гарантии, рассрочка
- `AUDIENCE_PAINS.md` — сегменты ЦА + матрица возражений
- `CASES_RESULTS.md` — 15+ кейсов учеников 1-го потока
- `SALES_MATCHMAKING.md` — запрос клиента → готовая связка (модуль + сотрудник + кейс)
- `SUCCESSFUL_DEALS.md` — анатомия 4 закрытых сделок (Vitalii / Vladimir / Yuliia / Олександр)
- `TELEGRAM_PLAYBOOK.md` — особенности ТГ-переписки (дробление, эмодзи, тайминги, follow-up)
- `PAYMENT_AND_ONBOARDING.md` — оплата (моно/50-50/крипта), сбор данных, передача куратору

## Как добавить свой продукт

1. Создай подпапку: `case-studies/<твой-продукт>/`
2. Возьми за образец структуру `kovcheg/`
3. Заполни:
   - `PRODUCT_OVERVIEW.md` — что продаёшь, сколько, что входит, гарантии
   - `AUDIENCE_PAINS.md` — кто покупает, какие боли, типовые возражения
   - `CASES_RESULTS.md` — реальные кейсы клиентов (если есть)
4. Опционально: если есть особенности канала (Telegram / WhatsApp / email) — добавь свой playbook
5. Скилл автоматически подхватит твою папку
